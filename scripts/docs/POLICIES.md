## Git管理ポリシー

### コミット対象

**✅ 必ずコミット:**
- `.tex` ソースファイル
- `.pdf` 生成ファイル（配布・プレビュー用）
- `.ps1` スクリプトファイル
- `.md` ドキュメントファイル
- `.toml`, `.json` 設定ファイル

**理由:**
- PDFは問題集として即閲覧可能、GitHub上でプレビュー可能
- ソースやスクリプトは再現性の中核

### コミット除外（.gitignore）

**❌ 除外対象（再生成可能）:**
- `.log` - LaTeX コンパイルログ
- `.aux` - LaTeX補助ファイル
- `.synctex.gz` - SyncTeX同期ファイル
- `.fls`, `.fdb_latexmk` - latexmk関連ファイル
- `.out`, `.toc` - LaTeX目次・参照ファイル

**理由:**
- プロジェクト固有の中間ファイル
- 再度コンパイルすれば自動生成される
- リポジトリサイズ削減

**過去遺産対応:**
- GitHubに既に追跡されていた中間ファイルは削除済み（2026-03-01）
- `git rm --cached` で Git追跡を除外

## ファイル組織化

### Scripts フォルダの原則

```
scripts/
├── docs/              # ドキュメント（このフォルダ）
│   ├── STRUCTURE.md          # プロジェクト構成
│   ├── HANDOVER_*.md         # 引き継ぎメモ
│   ├── POLICIES.md           # このファイル（運用ルール）
│   └── ... その他ドキュメント
│
├── compile/           # LaTeX コンパイル用スクリプト
│   ├── compile_all.ps1       # 全TeXファイルをコンパイル
│   ├── compile_all_standalone.ps1
│   └── compile_all_tikz.ps1
│
└── migration/         # マイグレーション・ツール
    ├── *.ps1                 # マイグレーション用スクリプト
    ├── *.md                  # 作業レポート・詳細記録
    └── ... ツール・ユーティリティ
```

**原則:**
- 一括処理スクリプトは **必ず** `scripts/` 配下に保存
- 問題・解答の本体 `.tex` は `university_exam/` に置く
- `scripts/docs/` には、方針・ルール・ハンドオーバーを置く

## ドキュメンテーション

### ドキュメント保存ルール

- **変更ログ・ハンドオーバー:** `scripts/docs/HANDOVER_*.md`
  - 新しい引き継ぎメモは日付付きで保存（例: `HANDOVER_2026-03-01.md`）

- **プロジェクト構成・技術仕様:** `scripts/docs/STRUCTURE.md`
  - ファイル命名規則、TikZ ライブラリ、コンパイル規約

- **運用ポリシー・ルール:** `scripts/docs/POLICIES.md`
  - このファイル。Git管理、チャット割り当て、ドキュメント原則など

- **スクリプト説明:** `scripts/README.md`
  - フォルダ構造、各スクリプトの概要

### ドキュメント更新タイミング

- **即時更新：** Git管理ポリシー・運用ルール変更時
- **月次更新（推奨）:** 大きな作業後、手作業による整理
- **新しい引き継ぎ発生時：** `HANDOVER_YYYY-MM-DD.md` を新規作成

## チャット・コンテキスト管理

### リポジトリごとチャット割り当て方式

**原則：リポジトリごとに専用チャットを用意**

**メリット:**
- ✅ コンテキストが明確に分離される
- ✅ 作業履歴がリポジトリごとに追跡される
- ✅ 長期的に管理・保守しやすい
- ✅ 各リポジトリの独立性が保たれる

**例:**
- `science-db` チャット ← このプロジェクト用
- `math-db` チャット ← math-db専用
- `dotfiles` チャット ← dotfiles専用
- その他のリポジトリ用チャット

### 統一ポリシー適用時

**複数リポジトリに同じ変更を加える場合:**

1. **各リポジトリのチャットで個別実施を推奨**
   - 各リポジトリの履歴に作業記録が残る
   - リポジトリの自律性を保つ

2. **例外：ポリシー定義自体が中央管理の場合**
   - `dotfiles` または `science-db` でルール化
   - 他リポジトリはそれを参照する設計

## Git運用フロー

### コミット範囲の判断

| 変更内容 | コミット | 例 |
|---------|---------|-----|
| ソース追加・修正 | ✅ | `.tex` ファイル修正 |
| PDF再生成 | ✅ | LaTeX コンパイル結果 |
| スクリプト追加・修正 | ✅ | `.ps1` ファイル新規作成 |
| ドキュメント更新 | ✅ | `.md` ファイル修正 |
| 中間ファイル更新 | ❌ | `.log`, `.aux` の変更 |
| OS/エディタ関連 | ❌ | `.DS_Store`, `Thumbs.db` |

### よいコミットメッセージ

**Conventional Commits ベース:**

```
feat:    新機能追加
fix:     バグ修正
refactor: コード整理（機能変更なし）
docs:    ドキュメント更新
ci:      CI/CD関連
chore:   依存関係やビルド設定
```

**例：**
```
feat: Add migration script for circuit-basics reorganization

- Create migrate_circuit_basics.ps1
- Move fig_em_cb_* files to new structure
- Update parent .tex file references
```
